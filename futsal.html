Introducción

    Propongo una aplicación para teléfonos inteligentes Android con acceso a bases de datos. Descarto otras soluciones como páginas web por cuestiones de rendimiento: una aplicación va a funcionar de mucho mejor manera que una página web en todos los dispositivos Android.

Módulos

    Tendríamos 3 módulos fundamentales. Por un lado, el que corresponde a quién está a cargo de mantener todos los datos de la aplicación actualizados, que además es el único capaz de cargar datos (nuevos equipos, nuevos jugadores de equipos, etc.) y modificar lo que considere necesario. Además, es el único capaz de crear perfiles de usuario dentro del sistema. Éste módulo corresponde al administrador. Por otro lado, a quienes están a cargo de comunicar los resultados de los partidos al administrador, a quienes llamaremos planilleros. Y por último, quienes ingresan a la aplicación en busca de la información cargada, a quienés llamaremos visitantes. Para diferenciar entre administrador y planilleros, se implementa un sistema de inicios de sesión con usuario y contraseña.
     Un administrador podrá cargar los resultados todo dato requerido y relacionado a una fecha, plantillas completas de equipos con la información deseable de cada jugador, y será capaz de modificar estos datos en todo momento. Un administrador puede acceder a un panel en donde le llegará la información proporcionada por los planilleros. Sólo un administrador podrá crear perfiles de usuario dentro del sistema, para por ejemplo, asignar un nuevo administrador.
     Un planillero podrá comunicar los datos de las planillas corespondientes a las fechas a un administrador. Tal vez y por lo hablado, mediante una fotografía.
     Un visitante podrá ver la información de cada partido disputado de las diferentes categorías contemplads.
     Una vez la aplicación cumpla con las funciones anteriormente mencionadas, diremos que la aplicación alcanzó el estandar de producto base.

Modo de trabajo

    El trabajo se desarrollará en base a incrementos, hasta alcanzar el producto base. Estimo 2 incrementos. Cada incremento supone una reunión en la que te muestro avances y me das un feedback "seguí por ahí", "no me gusta eso", "esto sería mejor si fuese de tal manera", etc.

Condiciones

         El trabajo empieza con un adelanto del %20.
     Mantenimiento: ante cualquier inconveniente estaré a disposición. El mantenimiento que ofrezco es estar dispuesto a presupuestar nuevos incrementos en la aplicación: "quisiera agregar cierta característica", "quisiera crear otro tipo de usuario de la aplicación", etc.
     Al ser una aplicación que requiere conectividad a bases de datos de internet, se terciariza este servicio. Los costos post producción de este servicio quedan al cargo del comprador. Disponer de esta aplicación de este calibre implica un costo mensual de alrededor de $400 a $700 en hosting para la base de datos. Probablemente, voy a terciarizar con una empresa argentina de Buenos Aires, "WNPower".
